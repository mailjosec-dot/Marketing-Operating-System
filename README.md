# Marketing OS

This repository is the source of truth for building, launching, and improving marketing systems across multiple brands and domains.

It is designed to give AI agents and human collaborators the same operating context:

- how we think
- how we write
- how we research customers
- how we create offers
- how we build funnels
- how we review quality
- how we document repeatable work

## Repository Map

| Folder | Purpose |
| --- | --- |
| `00_SYSTEM` | Core instructions, writing rules, brand principles, and decision rules. |
| `01_KNOWLEDGE_BASE` | Permanent marketing knowledge library. |
| `02_FRAMEWORKS` | Reusable templates for offers, funnels, brands, and campaigns. |
| `03_PLAYBOOKS` | Step-by-step execution guides for recurring work. |
| `04_DOMAINS` | One folder per business, product, or domain. |
| `05_ASSETS` | Reusable creative and brand assets. |
| `06_SOPS` | Standard operating procedures. |
| `07_QA` | Checklists used before any deliverable is considered complete. |
| `08_ARCHIVE` | Retired ideas, old experiments, and past versions. |
| `09_PRDs` | Product requirements documents for larger builds. |

## How To Use This Repo

1. Start every new project with a PRD in `09_PRDs`.
2. Create or update the domain folder in `04_DOMAINS`.
3. Use the relevant framework from `02_FRAMEWORKS`.
4. Follow the matching playbook in `03_PLAYBOOKS`.
5. Review the work against `07_QA`.
6. Move old or rejected work into `08_ARCHIVE` instead of deleting it.

## Working Standard

Every deliverable should be clear, specific, human, and useful. If it sounds generic, inflated, or like AI, revise it.
